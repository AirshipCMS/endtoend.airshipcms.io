1. Check Resources Loaded over http://

If you are loading any javascript with an "http://" protocol, your scripts will not load over "https://". Please update your code to load over "https://" or use // to allow either protocol (recommended).

2. Check API Calls

Check your scripts. Be sure to update "http://" Airship API calls to "https://" or use the relative path /api/* to use a relative protocol (recommended).

3. Check Image & List of Images Fields

You may notice "Mixed Content" warnings for images with an "http://" url. If you are using propellers within Airship templates, you can ensure that secure versions of images are loaded by using {{secure_url}} instead of {{url}} wherever image and list of images fields are used.

On your site, I noticed that the only place where you use {{url}} is on the `/funding` page. You may want to update the reference.

Image resources will still load with an "http://" url when "Mixed Content" warnings show in the console, so updating your templates and layouts to secure urls is highly recommended, though not required.

4. Check hardcoded assets in Styles, Templates, & Layouts

If you have any hard-coded image tags in your markup that are using absolute Cloudinary urls, update them to "https://" URLs.

5. Check CMS Richtext & Link, and List of Links Fields

Be careful, there is no "secure_url" for Link and List of Link fields. If you want to ensure that any Cloudinary resource URLs for link fields are https:// you must update the url in the CMS.

I have manually checked the URLs on your site, and all published instances of http:// cloudinary URLs have already been updated to https://. Please also check once more, to ensure that no URLs have been missed.
